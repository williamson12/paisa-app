1. APPLICATION OVERVIEW
--------------------------------------------------------------------------------
Purpose: 
Paisa is a premium personal finance tracker tailor-made for the Indian market. It 
focuses on minimalist, high-end UI/UX (similar to CRED/Apple) and provides AI-powered 
financial advice (CA Advisor) using Google's Gemini API.

Core Features:
- Real-time expense and income tracking.
- Google Authentication (Firebase).
- Monthly budget management and progress tracking.
- Categorized spending insights with Recharts.
- Indian-context AI financial advisor (SIP, PPF, Section 80C advice).
- Data synchronization between Firestore and LocalStorage for offline capabilities.

Tech Stack:
- Frontend: React 19, Vite 8, Tailwind CSS 4.
- Logic: Functional Components with Hooks.
- Styling: Vanilla CSS (embedded) + Tailwind 4.
- Icons: Lucide React (integrated via SVG paths in code).
- Animations: Framer Motion.
- Backend/DB: Firebase Auth & Firestore.
- AI Layer: Google Gemini 1.5 Flash.

2. FILE-BY-FILE BREAKDOWN

3. DATA FLOW & ARCHITECTURE
--------------------------------------------------------------------------------
Data Flow:
1. User interacts with UI (e.g., adds a transaction).
2. App state updates (`setData`).
3. Side-effect (`save` function): 
   - Async write to Firestore (`setDoc`).
   - Sync write to `localStorage` (`STORAGE_KEY`).
4. Read Flow: `onSnapshot` listens to Firestore. If Firestore is slow/offline, 
   a 4s timeout triggers a fallback to `localStorage`.

API Structure:
- Firebase Auth: SDK-based user management.
- Firestore: NoSQL document-based storage (`appData/{userId}`).
- Gemini API: Direct HTTPS POST fetch from the frontend to Google's Generative Language API.

Authentication:
- Uses GoogleAuthProvider via Pop-up.
- `onAuthStateChanged` manages session persistence.

5. BUGS & ERRORS (CRITICAL)
--------------------------------------------------------------------------------

ISSUE 1: Google Sign-In Fails on Mobile Browsers
- Location: src/App.jsx (handleGoogleSignIn)
- Problem: `signInWithPopup` is often blocked by mobile browser popup blockers.
- Root Cause: Reliability on popup-based auth.
- Fix: Implement `signInWithRedirect` as a fallback or primary method for mobile.

ISSUE 2: Gemini Advisor API Failure
- Location: src/App.jsx (go function)
- Problem: "No response generated" or CORS errors.
- Root Cause: Direct client-side fetching exposes the request to CORS policies 
  and exposes the API key.
- Fix: Use a backend proxy (Netlify Function) to hide the key and handle the request.

ISSUE 3: Monolithic App Structure
- Location: src/App.jsx
- Problem: 1000+ line file is difficult to maintain.
- Root Cause: Components and CSS are in one file.
- Fix: Componentize into a `/src/components` directory.
